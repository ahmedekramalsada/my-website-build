## Coding Rules

1. **Roadmap First** — Before writing any code, create a clear step-by-step plan for how you will complete the task. Share this plan with me before proceeding.

2. **Test Before Moving On** — After completing each step, test it to confirm it works correctly. Do not move to the next step until the current one is verified and working.

3. **Small, Verified Increments** — Never write large blocks of code all at once. Work in small, testable chunks. Each chunk must be confirmed working before continuing.

4. **No Broken Code** — Do not deliver or continue building on top of code that has errors or untested logic. Fix issues at each step before proceeding.
